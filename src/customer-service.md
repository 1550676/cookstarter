### Customer service API

**Проверка токена**

POST /token/check - путь поправим при необходимости, когда будет готов сервис авторизации
```json5
{
     "token": "test-token", //обязательно
}
```
ответ
```json5
{
    "status": "ACTIVE"
}
```
статусы:
- ACTIVE - действительный токен
- MISSED - недействительный токен
